# 简报风格改进设计

## 目标

让 AI 生成的简报更自然、更有可读性，像朋友聊天分享新闻，而不是写论文摘要。

## 问题诊断

当前简报的问题：
1. **Insight 公式化**：每条都是"核心发现是X；相比Y，它Z" — 读一篇知道后面全部
2. **Summary 八股化**："该主题聚焦...反映了...对...有参考价值" — 典型的官腔报告
3. **结构单一化**：永远是"主题→大段总结→文章列表" — 毫无变化

## 改进方案

### Prompt 重写

修改 `internal/service/briefing_service.go` 中的 `buildPrompt` 函数，核心改动：

1. **加入 Few-Shot Examples**
   给出 1-2 个完整示例，展示"好简报"长什么样

2. **明确禁止公式化表达**
   - 禁止："核心发现是..."
   - 禁止："相比XXX，它YYY"
   - 禁止："对XXX有参考价值"
   - 禁止："该主题聚焦..."

3. **正向引导**
   - 像朋友聊天一样分享
   - 用"有意思"、"值得关注"、"踩坑"等口语化表达
   - 每条 insight 聚焦一个点，不重复

4. **Topic Summary 要求调整**
   - 去掉"该主题聚焦"开头
   - 允许长短不一（有些 topic 一句话就够）
   - 鼓励观点和判断，不只是描述

### 示例风格

**Before（当前）：**
```
核心发现是一款游戏化的GPU架构学习工具上线；相比传统技术文档，它以互动形式降低专业门槛...
```

**After（目标）：**
```
有个游戏工具可以让你边玩边学 GPU 架构，之前只有干巴巴的文档。挺有意思的。
```

## 实现细节

### 文件改动

- `internal/service/briefing_service.go`:
  - 重写 `buildPrompt` 函数
  - 添加 few-shot examples 在 prompt 中
  - 更新输出格式说明（保持 JSON 结构但调整写作要求）

### Prompt 结构调整

```go
return fmt.Sprintf(`System: 你是一个科技新闻爱好者，给朋友分享今天看到的有趣内容。

要求：
- 口语化，像聊天
- 每条 insight 说清楚"是什么"和"为什么值得看"，不要废话
- 禁止：核心发现、相比XXX、对XXX有参考价值、该主题聚焦
- Topic summary 要有观点，不是罗列

示例：

Topic: AI 开发工具新动态

Summary: 今天看到几个有意思的工具——有个游戏化学 GPU 学习的，有共享 GPU 节点的 sllm，还有一个编码 Agent 的框架整理。总体感觉是：门槛在降低，边界在扩大。

Articles:
- 一个游戏化的 GPU 架构学习工具，互动式的，比看文档有意思多了
- sllm：共享 GPU 节点，便宜无限制，OpenAI 兼容 API

输出格式（严格 JSON）：
{
  "topics": [
    {
      "name": "主题名称",
      "summary": "口语化的主题总结，有观点，不罗列",
      "articles": [
        {"id": 1, "insight": "一句话说明亮点，像朋友聊天"}
      ]
    }
  ]
}

...
`)
```

2. Summary 读起来像聊天还是报告？
3. 风格是否统一？
